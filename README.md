# Climate-driven synchronization of wind–solar shortfalls

Code for reproducing the results in the manuscript:

> Climate-driven synchronization of wind–solar shortfalls challenges
> China's transition toward a carbon-neutral power system.

## Requirements

- Python ≥ 3.9
- NumPy, SciPy, Pandas, Matplotlib, openpyxl
- HiGHS solver (via `scipy.optimize.linprog`)

```bash
pip install numpy scipy pandas matplotlib openpyxl
```

## Data availability

Processed intermediate data for reproducing figures are included in
`data/processed_results/`. Raw third-party inputs are not redistributed:

- **CMIP6**: Available from [ESGF](https://esgf-node.llnl.gov/)
- **ERA5/ERA5-Land**: Available from [Copernicus CDS](https://cds.climate.copernicus.eu/)
- **Capacity pathways**: Supplementary materials of [Zhuo et al. (2022)](https://doi.org/10.1038/s41467-022-30747-0)
- **Transmission network**: Supplementary materials of [Shen et al. (2025)](https://doi.org/10.1038/s44221-025-00408-9)

## Quick start

From the package root:

```bash
python code_submission/plot_manuscript_figures.py
```

This regenerates the manuscript plotting set that is covered by package-local
plotting scripts. Figures are written to `figures/`.

## Full analysis scripts

The `data_generation_*.py` scripts document the full upstream analysis chain:

1. Quantile-delta mapping and capacity-factor correction
2. Climate-driven demand adjustment
3. Supply-demand matching
4. LSLW event identification
5. Provincial dispatch and transmission analysis
6. Attribution and provincial typology

These full-generation scripts require raw ERA5, ERA5-Land, CMIP6 and
third-party power-system input files that are not redistributed in this package.
Package-local paths and optional environment variables for raw inputs are
defined in `path_config.py`.

## Notes

- Final manuscript figures are in `figures/`.
- Source-data tables are in `data/source_data/`.
- Processed intermediate results are in `data/processed_results/`.
- A figure-to-data mapping is provided in `docs/figure_source_mapping.csv`.
- The 5-model ensemble consists of: ACCESS-CM2, EC-Earth3, GFDL-ESM4, MRI-ESM2-0 and NorESM2-MM.
- Package-local paths and optional environment variables for raw inputs are defined in `path_config.py`.
